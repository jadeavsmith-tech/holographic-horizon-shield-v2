## 🚀 The Pull Request (PR) Workflow

To contribute code updates safely from your terminal setup:
1. **Fork the Repository:** Create a local development fork of the repository.
2. **Isolate Your Feature:** Work inside a distinct, clearly named local branch (e.g., `feature/vector-matrix-calculations`).
3. **Validate Locally:** Execute tests using the `Red_team_mode.py` validation script to confirm that your updates do not break the 9.62 event horizon intercept mechanics.
4. **Submit a Clean PR:** Open a pull request against our `main` branch, explicitly citing the specific issue number you are resolving.
